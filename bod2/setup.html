i全体像と仕組み

メンバー専用ページの「スケジュール表」「進捗チェック」「ビジター一覧」など 全セクション が、Google Sheet の対応するタブに連動します。

┌──────────────────┐      ┌──────────────────┐      ┌──────────────────┐
│  メンバーが       │      │  Google          │      │  members.html    │
│  シートを編集     │ ───▶ │  Sheets          │ ───▶ │  ページが開く     │
│ （担当範囲のみ）  │      │ (gviz JSON API)  │      │  たびに自動取得   │
└──────────────────┘      └──────────────────┘      └──────────────────┘
サーバー不要・無料。Sheets の gviz JSON API を使うので、GitHub Pages のままで動きます。

1Google Sheet を新規作成

  1. https://sheets.new を開く（新規シートが立ち上がります）
  2. ファイル名を BOD 2026 第2回 メンバー専用データ など分かりやすい名前に変更

211 個のタブを作成して CSV を読み込む

下の表の タブ名 でシート下部のタブを作成し、それぞれに対応する CSV を読み込みます（タブ名は 必ず半角小文字、英字スペルが一致しないと取得できません）。

2-1. 各タブの追加方法

  1. シート下部の + ボタンで新規タブを追加
  2. タブ名をダブルクリックして変更（例: schedule）
  3. そのタブを表示した状態で ファイル → インポート → アップロード から、対応する CSV を選択
  4. インポート設定を 「現在のシートを置換」 にして「データをインポート」

2-2. 用意済みの CSV テンプレート

下のリンクから 11 個の CSV をダウンロードし、各タブに順に取り込んでください。

schedule.csv tasks.csv announcements.csv updates.csv roster.csv links.csv visitors.csv venue.csv equipment.csv documents.csv faq.csv
最初の自動生成された「シート1」（空の標準タブ）はあとで削除して構いません。

3共有設定（読み取り用）

ページがシートを読み込めるよう、「リンクを知っている全員：閲覧者」 に設定します。

  1. 右上の 共有 ボタンをクリック
  2. 「リンクを知っている全員」を選択
  3. 権限を 閲覧者 に設定 → 「完了」
シートは「閲覧者」公開ですが、メンバー専用ページ自体が noindex + 非リンク運用なので、URLを知っている人だけがアクセスする運用です（ページとシートで同じセキュリティ水準）。

4メンバーを編集者に追加・セクション別保護範囲

編集できるメンバーを追加し、タブごとに編集できる人を絞り込みます。

4-1. 編集者を追加

  1. 右上 共有 → メンバーのメールアドレスを入力 → 権限を 編集者 に
  2. 運営チーム全員、または対象メンバーのみ追加

4-2. タブ別に編集権限を分ける（推奨設定）

シートメニュー データ → シートと範囲を保護 から、タブ単位で「編集できる人」を制限できます。

タブ編集できる人（推奨）備考
schedule運営チーム全体スケジュール、本番までの予定
tasks運営チーム進捗・タスクの追加と完了マーク
announcements会長 / 副会長 / 広報担当連絡事項の掲示板
updates運営チーム変更履歴のログ
roster会長 / 副会長 のみ役員・連絡先（個人情報）
links運営チームグループLINEなどのリンク
visitors全メンバー各自のビジター招待状況を更新
venue会場担当会場・利用時間・連絡先など固定情報
equipment会場担当備品リストと準備状況
documents運営チーム資料・議事録のリンク集
faq運営チームよくある質問の追加・更新
保護されたタブを編集しようとすると、Google Sheets が「権限がありません」と表示します。手順としては、「保護したいタブを表示 → データ → シートと範囲を保護 → 「シートを追加」→ 「権限を設定」→ 自分＋指定メンバーのみに編集を許可」です。

5シートID を members.html に貼り付け

5-1. シートIDをコピー

シートのURLを見ると、こんな構造になっています。

https://docs.google.com/spreadsheets/d/1abc...XYZ123/edit#gid=0
                                       ↑↑↑ ここがシートID（44文字くらい） ↑↑↑

5-2. members.html を編集

docs/bod2/members.html をテキストエディタで開き、ファイル末尾近くの CONFIG ブロックを編集します。

const CONFIG = {
  // ↓ ↓ ↓ ここに貼り付け
  SHEET_ID: '1abc...XYZ123',

  REFRESH_INTERVAL_MS: 0,  // 例: 5分自動更新なら 300000
  ...
};

5-3. GitHub にコミット & プッシュ

  • GitHub Desktop / コマンドラインで変更をコミット → プッシュ
  • GitHub Pages が自動でデプロイ（数十秒〜数分）

6動作確認 & ふだんの運用

6-1. 確認方法

  1. members.html を開く
  2. ページ上部の「同期バー」が 緑色 ✓ シート同期 OK になればOK
  3. 各セクションの右上に「✎ シートで編集」リンクが表示されているはず

6-2. メンバーの操作

あとはメンバーが Sheet を直接編集するだけ。ページを開けば最新データが表示されます。

  • シート編集 → ページ再読み込み（または「再読み込み」ボタン） → 反映
  • 「✎ シートで編集」リンクをクリック → 該当タブが直接開く
  • LINE グループに シートのURL と members.html のURL を固定メッセージで置いておく運用がおすすめ

6-3. 自動更新を入れたい場合

REFRESH_INTERVAL_MS: 300000（5分）などにすると、開きっぱなしのページが定期的に最新化されます。本番当日に使う場合に便利。

📋タブ仕様一覧（コピー用カラム名）

CSV テンプレートを使えば自動で正しい列構成になりますが、手動で再構築する時の参考に。1 行目をヘッダ行として、以下のカラム名（半角英小文字）にしてください。

tasks

announcements

連絡事項

date · level（important/info）· body

updates

更新履歴

date · change

roster

メンバー名簿（役割カード）

role · name · contact · note

links

グループLINEなどのリンク集

label · url · icon（LINE/URL など）

visitors

招待ビジター一覧

name · category · inviter · status（confirmed/pending/declined）

venue

会場情報（キー・バリュー形式）

key · value — キーは venue_name, venue_address, venue_access, venue_map_url, hours_text, hours_detail, load_in, contact_name, contact_tel, contact_line

備品リスト

FAQ

question · answer

?うまく動かないとき

同期バーが「✗ シートが読み取れません」

  • シートIDが正しいか確認（URLの /d/ と /edit の間）
  • 共有設定が「リンクを知ってる全員：閲覧者」になっているか
  • シートが新規作成直後（数秒待って再読み込み）

「⚠ 一部タブが取得できませんでした」

  • タブ名が 半角英小文字 で一致しているか（Schedule ではなく schedule）
  • CSVを取り込んだ際に 「現在のシートを置換」 を選んだか（追加だと列がずれる）
  • カラム名（1行目）が仕様通りか（上の タブ仕様一覧 参照）

ステータスバッジが「英語表示」になる

status 列の値が小文字英語（done, doing など）になっているか確認してください。日本語（"完了"など）を入れるとそのまま表示されます。

日付がうまく表示されない

日付列のセル形式は「日付」推奨。テキスト型でも 2026-08-26 形式なら認識します。

本番当日に開きっぱなしのページを最新化したい

members.html の CONFIG.REFRESH_INTERVAL_MS を 60000（1分）や 300000（5分）に変更。
